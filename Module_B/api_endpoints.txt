 

AUTHENTICATION ENDPOINTS
 

POST   /login
       Auth: No
       Usage: Login with username & password, returns JWT token
       Body: {"user": "username", "password": "password"}

GET    /isAuth
       Auth: Yes (JWT)
       Usage: Check if current session token is valid
       Returns: User info and token expiry

POST   /logout
       Auth: Yes (JWT)
       Usage: Logout and invalidate current JWT token

GET    /
       Auth: No
       Usage: Welcome message

GET    /ui
       Auth: No
       Usage: Web UI interface

 
MEMBERS ENDPOINTS (/members)
 

GET    /members/
       Auth: Login Required
       Usage: List all members (admins see full details, others limited)

GET    /members/<member_id>
       Auth: Login Required
       Usage: Get specific member details

GET    /members/portfolio/<member_id>
       Auth: Login Required (own only or admin)
       Usage: Get comprehensive member portfolio including:
              - Member details
              - Role type (player/coach/admin)
              - Teams (if player)
              - Player statistics (if player)
              - Attendance records (if player)
              - Equipment loans
              - Facility bookings
              - Complaints filed

POST   /members/
       Auth: Admin Only
       Usage: Create new member
       Body: {"Member_ID", "Name", "Gender", "Email", "Age", "Phone_Number"(opt), "DOB"(opt)}

PUT    /members/<member_id>
       Auth: Login Required (own profile or admin)
       Usage: Update member information
       Body: {"Name", "Email", "Phone_Number", "Age", "DOB"} (any subset)

DELETE /members/<member_id>
       Auth: Admin Only
       Usage: Delete member (cascade delete)

AUTHENTICATION LEVELS
 

No Auth     : Public endpoint, no authentication required
Login       : Requires valid JWT token in Authorization header
Admin Only  : Requires JWT token with admin role

 
REQUEST HEADERS
 

For authenticated endpoints:
  Authorization: Bearer <JWT_TOKEN>
  Content-Type: application/json (for POST/PUT requests)

 
COMMON RESPONSE CODES
 

200 OK              : Successful GET/PUT request
201 Created         : Successful POST request
400 Bad Request     : Missing/invalid parameters
401 Unauthorized    : Missing/invalid token
403 Forbidden       : Insufficient permissions
404 Not Found       : Resource not found
